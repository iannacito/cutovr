Related rows grouped safely

We found {{ preview.merged_groups|length }} group{% if preview.merged_groups|length != 1 %}s{% endif %} of PCLaw rows that balance together.

Some PCLaw reference numbers don’t balance on their own, but the related rows that share the same source-journal tag (for example GB, GJ, CER) do balance when posted together. We’ll send each of these groups to QuickBooks as one balanced journal entry. Your debits and credits still match dollar for dollar — nothing unsafe goes through.

{% for g in preview.merged_groups %} {% endfor %}
Source-journal tag PCLaw references Lines Debits Credits
{{ g.token }} {{ g.transaction_ids|join(', ') }} {{ g.line_count }} {{ g.debits }} {{ g.credits }}
Why are these grouped?

PCLaw sometimes splits one bookkeeping batch — payroll, for example — across several reference numbers, with the offsetting side on a different reference. Looked at one reference at a time, the entries look unbalanced. Looked at by source-journal tag (the first word in the memo, e.g. GB for General Bank), they balance to the cent. We only group rows that share a tag and that sum to zero debit/credit difference, so the result is always a valid QuickBooks journal entry.
