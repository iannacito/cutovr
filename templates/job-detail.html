Reversal

Reverse this import

A reversal posts offsetting JournalEntry records to QuickBooks. It does not delete the original entries — both sides remain visible for audit. Use this if you imported into the wrong period, the wrong company, or with the wrong account mapping.

In QuickBooks, reversals appear as new journal entries with DocNumber starting REV- and line descriptions prefixed REVERSAL. The Journal report will list them alongside the originals; QuickBooks does not mark the originals as voided.

Security

Security & compliance

  • Source file encrypted immediately after upload with AES-256.
  • Processed export encrypted before storage.
  • OAuth tokens encrypted at rest.
  • Access tokens refreshed automatically before expiration.
  • Tokens removed when job is deleted.

Danger zone

Delete local job data

Permanently deletes the encrypted source file, encrypted output file, the QuickBooks tokens for this job, and the local job record. This cannot be undone.

What this does NOT do

  • It does not delete or reverse any JournalEntry records already posted in QuickBooks. Those stay in your QBO company.{% if job.qbo_results %} If you want to remove the {{ job.qbo_results|length }} entry/entries posted by this job, use Reverse this import above first.{% endif %}
  • It does not remove this file from the duplicate-import history. If you upload the same file to the same QuickBooks company later, it will still be blocked as a duplicate — that protects you from accidental double-posting.
  • It does not remove audit-log rows for this job; those are kept for compliance.
